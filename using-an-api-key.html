using an API key

Now that you've learned how to use an API without an API key, here's a guide on using one with an API key.

Many APIs require that you use an API key, which allows the API to check who your request is coming from. For example, in paid APIs it can be used to check if you have paid for a certain plan of that API or to charge you depending on the number of requests you have made.

If you don't yet have an api key for the cat api, head over to this page. If you already have your API key, head over to your email and copy it onto your clipboard.

When using an API key, its really important to keep it a secret. One of the ways we can do this in public codebases (such as a public repository on GitHub) is use environmental variables.

This guide will cover using a .env file to store an API key for testing, then deploying on Vercel with an API key as an environmental variable.

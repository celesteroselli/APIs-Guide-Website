What even is an API?

API stands for Application Programming Interface and is something which enables systems to communicate.

Essentially what happens is your application will call an API, which takes your request and sends it to the server, then retrieves the data for you and returns the response.

As briefly mentioned on the home page, there are tons of cool APIs out there which allow you to retreive data like the weather, calendar data, or even logging data for machine learning model training using Comet.ml.

One example of an API is CatAPI, an API which allows you to retrieve the URLs for random cat images, which you can then display on webpages or in your own application.

API keys are required for a few non-paid APIs, especially for more advanced functionality, and always for paid APIs. They essentially allow the API to determine who is using it, which has a variety of uses from tracking a specific user's usage (to determine the amount to charge them from using the API, in the case of paid APIs) or restricting the number of requests a certain application can send to an API in a period of time.
